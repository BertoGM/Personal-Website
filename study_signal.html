About the Project

As a UW student, finding a good study spot on campus can be a guessing game — you show up to Odegaard and every seat is taken, or you walk all the way to the library and it's dead quiet. StudySignal fixes that.

It's a crowdsourced study spot tracker built specifically for the UW Seattle campus. Students can check real-time busyness levels reported by other students, or submit their own update — helping the whole campus make smarter decisions about where to study.

What It Does
  • ▸ Displays study spots around UW Seattle on an interactive Google Maps view
  • ▸ Live busyness indicators — Empty, Medium, or Busy — crowdsourced from users
  • ▸ Timestamp showing when each location was last updated
  • ▸ Location safety validation — new spots are checked to be within campus boundaries, preventing fake or off-campus submissions
  • ▸ Users can submit new study spots directly through the interface
How It's Built

The backend is fully serverless on AWS. API Gateway handles incoming HTTP requests and routes them to Lambda functions, which process busyness updates and location submissions. All data is stored in DynamoDB — a NoSQL database that scales well for real-time read/write patterns like this.

The frontend uses the Google Maps API to render the campus map and place study spot markers. The site is deployed on Vercel for fast, zero-config hosting.

Known Issues & Limitations
  • ⚠ Student verification not yet implemented — anyone can currently submit updates. A future version will verify UW student status to improve data reliability.
  • ⚠ Data not always refreshing correctly — busyness updates occasionally require a manual page reload to reflect the latest state.
  • ⚠ UI/UX still being refined — the interface works but needs polish, better mobile responsiveness, and clearer visual feedback on interactions.
What's Next

The biggest planned improvement is UW student verification — likely using UW SSO or email domain validation — so busyness data comes from actual students rather than anonymous users. Other planned work includes fixing the data refresh bug, improving the mobile UI, adding automatic expiry for stale busyness reports, and potentially expanding to other UW campuses.
