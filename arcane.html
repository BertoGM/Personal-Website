Mobile Game

Arcane Dungeon

Kotlin · Jetpack Compose · Android

In Development
About the Game

Arcane Dungeon is a mobile roguelike dungeon crawler where every run is different. Players descend into procedurally generated dungeons, battling enemies with magic-based combat, discovering new spells, and pushing deeper with each attempt.

The game was built entirely in Kotlin using Jetpack Compose for the UI — a deliberate choice to work with modern Android development patterns rather than a game engine. The focus has been on smooth mobile-first controls and a satisfying gameplay loop that rewards both quick decisions and long-term build strategy.

Core Gameplay
  • ▸ Procedurally generated dungeon rooms — no two runs are the same
  • ▸ Magic-based combat system with a growing spell library
  • ▸ Randomized loot — collect spell scrolls, upgrades, and artifacts
  • ▸ Escalating dungeon difficulty across floors
  • ▸ High replayability through item and spell synergies
  • ▸ Responsive UI designed for one-handed mobile play
Key Features
Procedural Generation

Rooms, enemy placement, and item drops are randomized each run using seeded generation — ensuring replayability while keeping layouts fair and navigable.

Spell System

Players collect and equip spells ranging from direct damage to area effects and utility. Spell combinations create emergent strategies that vary by run.

Mobile-First Controls

Built from scratch for touch — directional controls, spell casting, and inventory management are all designed for comfort on small screens without a controller.

Jetpack Compose UI

The entire game UI is built in Compose rather than traditional Android XML views — giving full programmatic control over animations, state, and layout transitions.

Tech Stack
Kotlin Jetpack Compose Android SDK Coroutines ViewModel State Management

Building a game in Compose rather than Unity or LibGDX was intentional — it forced a deeper understanding of state management, composable recomposition, and how to drive game loops within Android's lifecycle architecture.

What I'm Learning
  • → Designing game loops and entity state outside of a dedicated game engine
  • → Procedural content generation algorithms for dungeon layouts
  • → Balancing player progression and randomness for a satisfying difficulty curve
  • → Touch-optimized UX patterns specific to mobile gaming
Current Status

The core gameplay loop — dungeon navigation, combat, and spell collection — is implemented and playable. Current work is focused on expanding the spell library, balancing enemy difficulty scaling, and polishing the UI before a wider playtest. The goal is an Android release via the Play Store once the game reaches a stable beta state.
